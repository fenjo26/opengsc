# Собственная ботоферма дорвеев (Индексатор)

Модуль **Индексатор (Indexer)** позволяет развернуть собственную закрытую ботоферму дорвеев на своих дроп-доменах или новорегах для быстрой и бесплатной индексации целевых сайтов в поисковых системах.

---

## Как это работает

Приватный индексатор перенаправляет реальных пользователей на целевой сайт (Money Site), а поисковым роботам отдаёт оптимизированный «мусорный» контент (дорвеи) с перелинковкой на продвигаемые страницы:

1. **Добавление домена:** Вы привязываете домен дорвея в панели OpenGSC (например, `doorway.xyz`).
2. **Развёртывание скрипта:** Загружаете сгенерированный PHP-код в корневую папку дорвея как `index.php`.
3. **Настройка Wildcard DNS:** Настраиваете домен так, чтобы любые поддомены (например, `abc.doorway.xyz`, `def.doorway.xyz`) вели на этот скрипт.
4. **Контентная база:** Скрипт генерирует бесконечные динамические страницы, смешивая ключевые слова из словаря (Dictionary) и расставляя ссылки на продвигаемые страницы из очереди (Queue).
5. **Стимуляция обхода:** Роботы переходят с одной страницы/поддомена на другие. При каждом заходе лог отправляется в вашу панель OpenGSC, отражая детальную статистику сканирования.

## Точное отслеживание ботов (Double DNS Lookup)

Чтобы защитить вашу ботоферму от раскрытия конкурентами, ручной проверки модераторами и парсинга сторонними сервисами (которые любят притворяться поисковыми роботами, подставляя фейковые User-Agent), в скрипт встроен **механизм двойной верификации DNS (Reverse + Forward DNS lookup)**.

### Принцип работы верификации:

1. **Обратный DNS-запрос (Reverse DNS / rDNS):**
   При заходе робота скрипт определяет его IP-адрес и запрашивает хостнейм с помощью `gethostbyaddr($ip)`.
   * Настоящий **Googlebot** должен резолвиться в хостнейм, оканчивающийся на `.googlebot.com` или `.google.com`.
   * Настоящий **Yandexbot** — на `.yandex.ru`, `.yandex.net` или `.yandex.com`.
   * Настоящий **Bingbot** — на `.search.msn.com`.
   * Настоящий **Mail.ru Bot** — на `.mail.ru`.

2. **Прямой DNS-запрос (Forward DNS / double-check):**
   Чтобы злоумышленник не мог прописать фейковый PTR-запись на своём IP, скрипт берёт полученный хостнейм и резолвит его обратно в IP с помощью `gethostbyname($hostname)`.
   * Если итоговый IP **совпадает** с исходным IP посетителя, бот признаётся валидным.
   * Если проверка провалена, бот считается фейковым.

### Маршрутизация трафика:
* **Настоящие роботы** (`$is_bot = true`) видят динамически сгенерированный дорвей, оптимизированный под обход поисковиков.
* **Люди и фейковые боты** (`$is_bot = false`) перенаправляются по 302-редиректу на ваш целевой сайт (`REDIRECT_TARGET`), что делает дорвей невидимым для посторонних глаз.

## Пошаговая настройка ботофермы

### Шаг 1 — Добавление домена в OpenGSC
1. Откройте панель OpenGSC, перейдите во вкладку **Indexer** -> **Domains**.
2. Нажмите **Add Domain**.
3. Укажите имя домена (без `http/https`, например `my-doorway-network.xyz`), выберите шаблон (E-commerce / Directory / Blog) и укажите целевой сайт в поле **Redirect Target URL** (куда перенаправлять людей).
4. Нажмите **Create Domain**. Панель сгенерирует для домена уникальный API-ключ.

### Шаг 2 — Настройка DNS и сервера
1. Направьте домен на ваш хостинг/сервер, где работает Apache/Nginx + PHP.
2. Настройте **Wildcard DNS** (A-запись вида `*` pointing to server IP), чтобы абсолютно любые поддомены вели на ваш сервер.
3. Настройте веб-сервер так, чтобы все запросы (включая несуществующие поддомены и пути) перенаправлялись на файл `index.php`. 
   * **Пример для `.htaccess` (Apache):**
     ```apache
     RewriteEngine On
     RewriteCond %{REQUEST_FILENAME} !-f
     RewriteCond %{REQUEST_FILENAME} !-d
     RewriteRule ^(.*)$ index.php [L,QSA]
     ```

### Шаг 3 — Установка PHP-скрипта
1. Перейдите во вкладку **Settings** в разделе **Indexer**.
2. В поле **OpenGSC Public Deployment URL** укажите внешний адрес вашей панели OpenGSC (без слэша на конце, например `https://opengsc.mydomain.com`). Скрипт подставит его для отправки логов.
3. В выпадающем меню выберите ваш домен для автоматического заполнения API-ключа и URL редиректа.
4. Нажмите **Copy Code** и сохраните скопированный код как `index.php` в корневой папке вашего дорвея.

## Интеграция с Google Search Console (Отправка Sitemap)

Вы можете автоматизировать отправку страниц ваших проектов напрямую в очередь дорвеев:
1. Зайдите в любой сайт в основном разделе OpenGSC.
2. В блоке управления картами сайта (Sitemaps) появится панель **Private Bot Farm**.
3. Выберите домен дорвея и нажмите **Submit Sitemap URLs**.
4. Скрипт распарсит все ссылки из sitemap.xml и добавит их в очередь на перелинковку внутри дорвеев. При следующем обходе боты сразу начнут сканировать ваши новые целевые страницы.

## Панель Статистики (Statistics)

Дашборд статистики даёт полную сводку по сканированию вашей сети дорвеев за последние 30 дней:
* **Суммарные карточки:** Отдельный подсчёт для Google, Yandex, Bing, Mail.ru, других роботов (Other) и редиректов пользователей (Redirects).
* **Ежедневная детализация (Daily Breakdown):** Таблица, в точности повторяющая структуру профессиональных лог-панелей:
  `Date | Google | 304 | Yandex | 304 | Bing | Mail.ru | Other | Total`
  * **Колонка 304 (Conditional Crawl):** Показывает, сколько раз роботы Google и Yandex получали ответ `304 Not Modified` вместо полной загрузки страницы. Это говорит о том, что боты активно мониторят домен, экономя ваш краулинговый лимит.
