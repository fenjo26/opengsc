# Подключение Google Analytics 4 (GA4)

Вкладка **GA4** на карточке сайта показывает реальные данные из Google Analytics — Сеансы, Вовлечённость (Engagement Rate), Ключевые события и Доход — с графиком по дням и динамикой к предыдущему периоду.

Это **опциональная** функция. Дашборд и интеграция с Google Search Console работают без неё. Настройка разовая и занимает пару минут.

> Если на вкладке GA4 пусто, приложение само покажет пошаговый гайд и кнопки для включения API. Здесь — те же шаги подробнее, плюс разбор ошибок.

---

## Кратко

1. Включить два API в Google Cloud (один раз на проект).
2. Переподключить Google-аккаунты, чтобы выдать доступ к Analytics.
3. Убедиться, что у аккаунта есть доступ к ресурсу GA4 в самом Google Analytics.
4. На вкладке GA4 выбрать ресурс и нажать **Link Property**.

---

## Шаг 1 — Включить два Google API

GA4 использует API, которых нет у Search Console:

- **Google Analytics Data API** — отдаёт сами цифры (сеансы, доход и т.д.).
- **Google Analytics Admin API** — отдаёт список доступных ресурсов GA4 для выпадающего меню.

Открой **Google Cloud Console** в **том же проекте**, где находится твой OAuth Client ID (тот, что указан в `GOOGLE_CLIENT_ID`), и включи оба:

| API | Ссылка |
|---|---|
| Google Analytics **Data** API | https://console.cloud.google.com/apis/library/analyticsdata.googleapis.com |
| Google Analytics **Admin** API | https://console.cloud.google.com/apis/library/analyticsadmin.googleapis.com |

На каждой странице нажми **Enable**. Убедись, что в правом верхнем углу выбран нужный проект.

> **Важно:** API включаются на **проект**, а не на аккаунт. Достаточно одного раза — это покрывает все подключённые Google-аккаунты сразу. После включения подожди 1–2 минуты — Google пишет про распространение настроек (propagation).

---

## Шаг 2 — Переподключить Google-аккаунты

Чтобы читать GA4, приложению нужен OAuth-доступ `https://www.googleapis.com/auth/analytics.readonly`. Он добавлен в приложение, но у **уже подключённых** аккаунтов токены старые — без этого доступа. Их нужно переавторизовать.

1. Открой **Settings → My Google Accounts**.
2. Войди заново под нужным аккаунтом (повторная авторизация).
3. На экране согласия Google должна стоять галочка про **Google Analytics**.

Проверка: напротив каждого аккаунта в настройках виден статус **GSC ✓** и **GA4 ✓**. Зелёная галочка **GA4** означает, что доступ к Analytics выдан.

> На **свежей** установке этот шаг не нужен — доступ запрашивается сразу при первом входе. Переподключение требуется только при обновлении со старой версии.

---

## Шаг 3 — Дать аккаунту доступ к ресурсу GA4

Ресурс (property) появится в списке на вкладке GA4, только если подключённый Google-аккаунт имеет к нему доступ **в самом Google Analytics**. Доступ в Cloud Console и доступ к ресурсу — это разные вещи.

Если нужного ресурса нет в списке:

1. Открой [Google Analytics](https://analytics.google.com/).
2. **Администратор** (шестерёнка) → **Управление доступом к ресурсу** (Property Access Management).
3. Добавь email подключённого Google-аккаунта с ролью хотя бы **Читатель (Viewer)**.

---

## Шаг 4 — Привязать ресурс

1. Открой карточку сайта → вкладка **GA4**.
2. Выбери ресурс в выпадающем списке.
3. Нажми **Link Property**.

Привязка сохраняется **отдельно для каждого сайта**. Отвязать ресурс можно в любой момент ссылкой **Unlink GA4** — данные не удаляются, просто разрывается связь.

---

## Режим OAuth-приложения (Testing / Production)

Скоуп `analytics.readonly` относится к **sensitive**:

- **Testing** — работает только для аккаунтов из списка **Test users** на OAuth Consent Screen.
- **Production (Опубликовано)** — работает для любого аккаунта. Без верификации Google покажет экран «приложение не проверено» → «Перейти всё равно» — для self-hosted это нормально.

«Restricted»-проверка Google для `analytics.readonly` не требуется — это sensitive-, а не restricted-скоуп.

## Разбор ошибок

**`Google Analytics Admin API has not been used in project ... or it is disabled`**
Не включён нужный API. Вернись к Шагу 1 и включи **Data API** и **Admin API** в правильном проекте. Подожди 1–2 минуты.

**`Request had insufficient authentication scopes` / `insufficient authentication scopes`**
Аккаунту не выдан доступ к Analytics. Переподключи его (Шаг 2) — должна появиться **GA4 ✓**.

**Список ресурсов пустой, но API включены и GA4 ✓ горит**
У аккаунта нет доступа к ресурсу GA4. Выдай доступ в Google Analytics (Шаг 3).

**`PERMISSION_DENIED` при привязке конкретного ресурса**
Тот же случай: у аккаунта нет прав на этот ресурс. Проверь Property Access Management.

## Какие метрики показываются

| Метрика | GA4 API | Как считается |
|---|---|---|
| Сеансы | `sessions` | сумма за период |
| Вовлечённость | `engagementRate` | средняя за период (в %) |
| Ключевые события | `keyEvents` | сумма за период |
| Доход | `totalRevenue` | сумма за период |

Дельта `+N%` рядом с каждой метрикой — сравнение с предыдущим периодом такой же длины.
